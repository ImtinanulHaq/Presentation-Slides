PRESENTATION ARCHITECTURE PATTERNS FOR LLM TRAINING
====================================================

=== PATTERN 1: TECHNOLOGY/PRODUCT PRESENTATION ===
Input Topic: AI in Healthcare
Input Audience: Hospital Administrators
Input Tone: Professional
Input Content: "AI is revolutionizing healthcare through diagnostic accuracy, automating administrative tasks, and improving patient outcomes"

Expected Output Structure:
- Total Slides: 6-8
- Slide 1: Title Slide (AI in Healthcare: Transforming Patient Care)
- Slide 2: Problem Statement (Current Healthcare Challenges)
- Slide 3: Solution Overview (How AI Addresses Challenges)
- Slide 4-5: Key Benefits & Use Cases
- Slide 6: Implementation Strategy
- Slide 7: ROI & Metrics
- Slide 8: Call to Action

Bullet Guidelines:
- Each bullet max 12 words
- Action-oriented language
- Data/metrics where possible
- Address pain points directly

Visual Guidance:
- Icons: stethoscope, computer, chart, shield
- Symbols: arrows (upward for growth), check marks (success), warning signs
- Images: hospitals, doctors, data dashboards, patients

---

=== PATTERN 2: BUSINESS/SALES PRESENTATION ===
Tone: Persuasive/Casual
Content Characteristics: Problem → Solution → Benefits → Pricing → Call to Action
Slide Count: 5-7 slides typically

Structure:
1. Hook Slide (attention-grabbing problem)
2. Market Opportunity (why now?)
3. Solution Overview (your product/service)
4. Competitive Advantage
5. Business Model/Pricing
6. Traction/Social Proof
7. Call to Action

Bullet Formula:
[Benefit] + [How/Why] + [Result]
Example: "Reduce manual data entry by 80% using automation"

---

=== PATTERN 3: EDUCATIONAL/ACADEMIC PRESENTATION ===
Tone: Academic/Informative
Audience: Students, Researchers, Professionals
Slide Count: 8-12 slides

Structure:
1. Title + Learning Objectives
2-3. Context/Background
4-7. Main Content (break into logical sections)
8-10. Analysis/Results
11. Conclusions
12. Questions/Further Reading

Key Features:
- Citations/References
- Data visualizations
- Complex concepts broken down
- Clear definitions
- Section dividers

---

=== BULLET POINT OPTIMIZATION RULES ===

GOOD:
✓ "Reduce operational costs by 40% annually"
✓ "Automate recurring tasks in under 5 minutes"
✓ "Support 10+ integrations out of the box"

BAD:
✗ "We have a solution that can help you reduce costs and improve efficiency by implementing advanced technology systems"
✗ "Our platform provides comprehensive solutions"

FORMULA:
[Action Verb] + [Object] + [Metric/Benefit] = Strong Bullet
Examples:
- Achieve 3x ROI within first year
- Manage 500+ team members from single dashboard
- Integrate payments in under 15 minutes

=== TONE ADJUSTMENT GUIDELINES ===

PROFESSIONAL:
- Formal language
- Data-driven
- Business focus
- Icons: formal charts, business elements
- Colors: blue, gray, black

CASUAL:
- Conversational tone
- Relatable examples
- Light emojis where appropriate
- Icons: friendly, colorful
- Colors: vibrant, varied

ACADEMIC:
- Precise terminology
- Sources/citations
- Detailed explanations
- Icons: books, microscope, research
- Colors: neutral, professional

PERSUASIVE:
- Question-based openings
- Problem emphasis
- Solution highlight
- Icons: target, spotlight, growth
- Colors: bold, contrasting

---

=== CONTENT ANALYSIS RULES ===

If user input has:
- Long paragraphs → Break into 3-4 slides
- Multiple topics → Use section dividers
- No numbers/metrics → Suggest adding data
- Unclear target → Ask for clarification
- Weak opening → Create compelling hook slide
- No conclusion → Add clear CTA slide

Word Count to Slide Count:
- 100-300 words → 3-4 slides
- 300-600 words → 5-7 slides
- 600-1000 words → 8-10 slides
- 1000+ words → 10-14 slides

---

=== EXAMPLE TRANSFORMATION ===

RAW INPUT:
"We built a platform that helps companies manage their projects. It has features for task management, team collaboration, and reporting. Companies save time and money. We think many businesses need this."

OPTIMIZED OUTPUT:
Problem: Inefficient project management costs companies time and money
Solution: Unified platform for task management, collaboration, reporting
Benefits: Faster delivery, improved team alignment, 30% cost reduction
Target Users: Mid-market to enterprise companies
Unique Value: All-in-one solution with enterprise support

Slide Structure (6 slides):
1. Title: Project Management Made Simple
2. The Problem: Current Tools Leave Gaps
3. Our Solution: One Platform, Complete Control
4. Key Features: Task, Collaborate, Report
5. Real Results: 30% Cost Savings Achieved
6. Get Started: Free 30-Day Trial

=== VISUAL TONE CONSISTENCY ===

If Professional:
- Minimalist icon style
- Clean, simple symbols
- Professional photography
- Blue/gray/black palette

If Casual:
- Playful icon style
- Colorful symbols
- Relatable real-world images
- Vibrant colors

If Academic:
- Scientific icons (microscope, beaker)
- Educational symbols
- Data visualizations, charts
- Neutral, professional colors

If Persuasive:
- Bold, attention-grabbing icons
- Impactful symbols (target, arrow)
- Emotional, inspiring images
- High-contrast colors
